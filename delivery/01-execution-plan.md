# Декомпозиція та план виконання

> Як **2 dev + 0.5 designer + 0.5 analyst** валідують одну гіпотезу за **2 тижні**, не будуючи повний функціонал. Плюс — як решта фіч лягає в роадмеп паралельними треками.
> Матриця залежностей/паралелі — [02-feature-matrix](./02-feature-matrix.md).

## 1. Команда та принцип

| Роль | Завантаження | Зона |
|---|---|---|
| **Dev A** (frontend / loop) | FT | стрік, екран «Сьогодні», стани, анімації |
| **Dev B** (notifications / data) | FT | нагадування, аналітика, A/B-флаг, події |
| **Designer** | PT (0.5) | макети станів, копірайт, a11y |
| **Analyst** | PT (0.5) | pre-registration A/B, дашборд, guardrails |

**Принцип скоупу:** валідуємо **петлю**, а не продукт. Усе, що не впливає на сигнал D2 — ріжемо.

## 2. Скоуп MVP (фіча `001-daily-focus`)

**IN:** A/B-флаг; стрік (серія днів) + мікросвята 3/7; щоденне локальне нагадування + контекстний opt-in після першої перемоги; гуманний пропуск (freeze); події `streak_*`/`reminder_*` з тегом `variant`.
**OUT (поза 2 тижні):** серверні пуш-кампанії; складна гейміфікація (ліги/бали); персоналізація часу через ML; соцфункції. (Усе AI — окремий трек, не MVP.)

## 3. Два тижні — по ролях × тижнях (Gantt-зріз)

| | Тиждень 1 | Тиждень 2 |
|---|---|---|
| **Dev A** | стрік-логіка `computeStreak` + freeze; стрік-чіп/банер на «Сьогодні»; інтеграція `getVariant` (control ховає стрік) | мікросвята 3/7; гуманний пропуск; a11y + reduced-motion; QA-фікси, demo-build |
| **Dev B** | opt-in + вибір часу після першої перемоги; планувальник нагадувань (1/день); deep-link «Сьогодні» | події `streak_*`/`reminder_*` з `variant`; «тиха» поведінка після day_completed; опт-аут у 1 тап; офлайн |
| **Designer** | макети: стрік-чіп/банер, opt-in+time-picker, текст нотифікації; стани 3/7/freeze | a11y-ревʼю (контраст, не-лише-колір); полір анімацій; фінальні ассети |
| **Analyst** | **pre-register A/B**: гіпотеза, Primary (D2), guardrails, MDE, sample size; tracking-план + QA подій; дашборд D0→D2 | перевірка спліту 50/50 (no SRM); запуск на когорту; щоденний моніторинг guardrails; директивний рід D2 + нотатка ship/iterate/kill |

**Контрольні точки:** кінець Т1 — петля збирається на dev-біллді; середина Т2 — запуск на когорту; кінець Т2 — директивний рід + рішення.

## 5. Чесна оцінка потужності A/B

2 тижні достатньо, щоб **зібрати петлю й запустити експеримент**, але **повна статпотужність** на D2 для ефекту **+5 п.п.** — це ~5–6 тижнів (~1 250/арм). Для **+8–10 п.п.** вистачає 2 тижнів. Тому ціль 2 тижнів — **директивний рід (тренд) + чисте інструментування + guardrails**; рішення ship/kill — після добору потужності. (Деталі — [plan.md §4](../specs/001-daily-focus/plan.md).)

## 6. Портфельна декомпозиція (поза MVP — для роадмепу)

Після MVP — будуємо рештою треками, **паралельно** (хто/коли — у [матриці](./02-feature-matrix.md)):

| Трек | Фічі | Власник-кор | Може стартувати |
|---|---|---|---|
| **Foundation** | `000-core-infrastructure` | Dev A+B | першим (база; у тест-кейсі вважаємо наявним) |
| **Retention MVP** | `001-daily-focus` | Dev A+B | після Today/подій фундаменту ← **це наші 2 тижні** |
| **AI enabler** | `ai-personalization-context` | Dev B | паралельно з MVP (інші руки), блокує AI-фічі |
| **AI features** | `003`→`006`→`004`→`002`→`005` (за RICE-lite) | Dev B + Dev A (UI) | **після enabler**, паралельно між собою |
| **In-app guidance** | `007-product-tour` (не-AI) | Dev A + Designer | паралельно будь-коли (залежить лише від поверхонь) |

## 7. Ризики й буфер
- **8 кроків онбордингу** ↑ drop-off → per-step трекінг, ріжемо за даними.
- **Opt-in пушів (iOS)** ↓ reach → контекстний запит після першої перемоги.
- **Перенагадати** → guardrail opt-out/uninstall + ≤1/день + «тиха» поведінка.
- Буфер: ~15% часу Т2 на QA/фікси; Should-story (зміна часу, тиха поведінка) — перші кандидати на виріз, якщо не встигаємо.
