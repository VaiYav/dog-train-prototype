> Згенеровано: 2026-06-09 · Codebase: `.` (репозиторій ініціалізовано SpecKit; продуктового коду ще немає)

## Architecture Overview

Це **greenfield**: у репозиторії є лише SpecKit/Product Forge риштування (`.specify/`, `.product-forge/`, `specs/`, `docs/`), без застосункового коду. Тому замість аналізу наявного коду ми **фіксуємо цільову архітектуру фундаменту** — вона стане обов'язковими обмеженнями для `spec.md` і `plan.md`.

Для тест-кейсу демонстрований артефакт — **клікабельні HTML-прототипи (mobile-first)**; бекенд/нативна частина — концептуальні (майбутній прод). Цільова форма:
```
HTML-прототип (mobile-first, phone-frame ~390px)   ← що будуємо зараз
  ├─ «екрани»: онбординг(8) → профіль → план → «Сьогодні»(3 завдання) → прогрес → нагадування
  ├─ стан: localStorage (профіль, відмітки, прогрес) — офлайн-first відчуття
  ├─ імітована аналітика: trackEvent(name, props) → лог подій (схема нижче) = майбутній PostHog
  └─ імітація: rules-based генерація плану (JS), локальні нагадування (UI-банер), A/B-перемикач

Прод (майбутнє, поза скоупом прототипу):
  RN/Flutter ──► REST backend (template engine без AI/ML) ──► DB: dogs, plans, tasks, completions, events
```

## Codebase Constraints

> Обов'язкові патерни фундаменту — вони мають бути відображені у spec.md / plan.md.

| Обмеження | Джерело | Вплив на дизайн фічі |
|---|---|---|
| **Без кастомних AI/ML** для генерації плану | constraint завдання | план — лише rules-based із куратованих шаблонів за сегментами (порода-група × вік × ціль) |
| **Офлайн-first для відміток** | UX-research | відмітки пишуться локально й синхронізуються; не блокувати дію на мережі |
| **Аналітика з першого екрана** | потреба Фази 2 | кожна ключова дія = подія з консистентною схемою (`event`, `props`, `user_id`, `ts`) |
| **Контекстний запит пушів** | UX-research / opt-in | дозвіл на нотифікації — після першої перемоги, не на старті |
| **Privacy/PII** | загальне | мінімум персональних даних; узгодження зі store-політиками для домашніх тварин |
