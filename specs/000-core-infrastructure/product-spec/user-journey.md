# User Journey: DogTrain (базовий продукт)

> Feature: `000-core-infrastructure` | Persona: «Новачок-власник»
> Related: [Product Spec](./product-spec.md) | [Wireframes](./wireframes.md)
> Контекст: клікабельні HTML-прототипи, mobile-first.

Два ключові потоки фундаменту: **(A) Активація (Day 0)** і **(B) Повернення (Day 2)**. Саме на стику цих двох потоків живе проблема відтоку — потік B сьогодні «порожній» і його підсилює retention-MVP Фази 3.

---

## Journey A — Активація (Day 0): install → перша перемога

> Entry: відкриття після інсталу · Exit (успіх): перший `task_completed` · Ціль: <2 хв.

| Крок | Дія користувача | Відповідь системи | Емоція | Нотатки / drop-off |
|---|---|---|---|---|
| 1 | Відкриває апку | 2–3 value-екрани (не карусель фіч) | 😊 | `onboarding_started` |
| 2–9 | Проходить **8 кроків** (імʼя/фото → порода → вік → вага* → активність → ціль → контекст → підтвердження) | Прогрес-бар; одне рішення/екран | 😐 | `onboarding_step_completed{step}` — **головна зона drop-off**, кожен крок ≤10с |
| 10 | Бачить «Ваш план готовий» | Rules-based генерація <2с | 😊 | `plan_generated{tasks_total}`; trial стартує тихо, без картки |
| 11 | Потрапляє на **«Сьогодні» (3 завдання)** | Перше завдання — тривіальне («коротка прогулянка») | 😊 | дефолтний екран — не тижневий план |
| 12 | **Відмічає перше завдання** | Галочка + вібрація + прогрес-кільце | 😄 | **`task_completed` = ПЕРША ПЕРЕМОГА** (ключова подія активації) |
| 13 | Бачить контекстний запит пушів | «Нагадати про прогулянку?» з поясненням | 😐 | `notification_permission{granted}` — **після** перемоги, не на старті |

**Drop-off risk points:** кроки 2–9 (8 екранів), пауза на генерації, перше завдання. **Guardrail:** onboarding completion ≥75%, TTV <2 хв.

## Journey B — Повернення (Day 2): тут «ламається» утримання

> Entry: `app_opened{day_since_install=2}` · Exit (успіх): `task_completed` у D2 · **Це потік, де гине 80% користувачів.**

| Крок | Дія користувача | Відповідь системи | Емоція | Нотатки / drop-off |
|---|---|---|---|---|
| 1 | (Тригер повернення?) | Локальне нагадування про прогулянку/догляд | 😐 | **Слабка ланка фундаменту:** якщо пуш вимкнено — тригера нема → не повертається |
| 2 | Відкриває апку на 2-й день | Одразу «Сьогодні» з 3 завданнями (без повторного онбордингу) | 😊 | `app_opened{day=2}` |
| 3 | Дивиться на завдання | Прогрес учора видно; сьогодні — нові 3 | 😐 | якщо немає мотивації/винагороди — закриває |
| 4 | Відмічає завдання | Галочка + прогрес | 🙂 | `task_completed{day=2}` — **сигнал утримання** |
| 5 | (Спрацьовує нагадування догляду) | «Скоро щеплення / грумінг» | 😊 | `care_reminder_fired` — підсилює унікальну цінність |

**Чому тут відтік:** у фундаменті повернення тримається лише на нагадуванні + новому списку. Немає винагороди за повернення (стрік), денний фокус ще можна перевантажити. **Саме це лікує retention-MVP Фази 3** (Daily Focus + стрік + гуманні нагадування).

## Error / Edge Scenarios
- **Офлайн при відмітці:** записуємо локально (`localStorage`), синхронізуємо пізніше; UI не блокується.
- **Вихід посеред онбордингу:** зберігаємо крок, повертаємо на нього.
- **Рідкісна порода:** fallback на породу-групу/розмір — план усе одно генерується.
- **Trial завершено (повторний вхід):** показуємо paywall, але історія/прогрес лишаються видимими.
- **Пушів не дозволено:** показуємо локальні UI-нагадування при відкритті + м'який повторний запит пізніше.

## Journey Metrics
- **Entry A:** відкриття після інсталу · **Exit A:** перша перемога (`task_completed` D0) · **Час:** <2 хв.
- **Entry B:** відкриття D2 · **Exit B:** `task_completed` D2 · **Drop-off:** відсутність тригера повернення / перевантаження / відсутність винагороди.
