# Problem Statement: Daily Focus loop (retention MVP)

> **Product Forge — Phase 0 (стислий).** Discovery/research для цієї фічі = **Фаза 2, [01-churn-analysis.md](../../../docs/product/01-churn-analysis.md)** (5 гіпотез + RICE). Тут лише фіксуємо рамку обраної фічі.

## Проблема (в одному реченні)
Churned-когорта не повертається на Day 2–3, бо немає **тригера повернення** і **винагороди** за повернення — перший тиждень не завершується.

## Обрана фіча (RICE-переможець)
**Daily Focus loop** = H2 (розумні щоденні нагадування, RICE 1275) як ядро + мінімальний стрік з H1 як винагорода. Замикає петлю **тригер → дія → винагорода** на поверхні «Сьогодні» фундаменту.

## Гіпотеза (формат тест-кейсу)
> **Як** власник собаки, який легко закидає догляд після першого дня, **я хочу** щодня отримувати нагадування про конкретні завдання для мого песика й бачити, як росте моя серія днів поспіль (стрік), **щоб** мати привід і мотивацію повертатись щодня та завершити перший тиждень.

## Чому це валідовно за 2 тижні (без AI/ML)
- Поверхня «Сьогодні», події (`task_completed`, `app_opened`), сутності (`Completion`) і `getVariant` — **уже у фундаменті** (Consumer Contract).
- Стрік = детермінований підрахунок послідовних днів; нагадування = локальна нотифікація за розкладом. Жодного ML.

## Ключові ризики (з RICE Confidence-аналізу)
1. Стеля reach через opt-in пушів (особливо iOS) → контекстний запит після першої перемоги.
2. «Перенагадати» → роздратування/uninstall → **guardrail** на opt-out/uninstall + 1 нагадування/день.
3. Стрік-тиск (страх втратити серію) демотивує меншість → **гуманний стрік** (freeze/rest-day, без різкого обнулення).
