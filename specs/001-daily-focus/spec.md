## Гіпотеза (формат тест-кейсу)

> **Як** власник собаки, який легко закидає догляд після першого дня,
> **я хочу** щодня отримувати нагадування про конкретні завдання для мого песика й бачити, як росте моя серія днів поспіль (стрік),
> **щоб** мати привід і мотивацію повертатись щодня та завершити перший тиждень.

## Overview

### Why
Аналіз відтоку (Фаза 2) показав: churned-когорта не має **тригера повернення** (`notification_permission` низький, `app_opened` D2 ≈ 0) і **винагороди** за повернення. Day 2 — «день рішення». Daily Focus loop замикає **тригер (нагадування) → дія (завдання) → винагорода (стрік)**.

### What
Поверх «Сьогодні» фундаменту вмикаємо: (1) щоденне нагадування під план у зручний час; (2) гуманний стрік «днів поспіль» із мікросвятами; (3) усе під A/B-флагом `daily_focus` (control = фундамент без активного стріку/нагадування, variant = повна петля).

## Goals
- **Primary:** підняти **D2 retention** (повернувся + ≥1 завдання) variant vs control.
- Secondary: D7 retention, completion 1-го тижня, днів поспіль.
- **Non-goals (v1):** серверні пуш-кампанії; складна гейміфікація (ліги, бали); персоналізація часу нагадування через ML; соцфункції.

## User Stories

### Must Have (MVP)
- [ ] **US-1.** Як власник, я хочу після першої перемоги увімкнути щоденне нагадування й обрати час, щоб мати привід повертатись. **AC:** контекстний запит після першого `task_completed`; вибір часу (Ранок/День/Вечір/свій); подія `reminder_scheduled{time}`.
- [ ] **US-2.** Як власник, я хочу щодня отримувати нагадування під мій план, щоб не забути про догляд. **AC:** 1 локальне нагадування/день у обраний час; текст під завдання дня; відкриття → deep-link на «Сьогодні»; події `reminder_fired`, `reminder_opened`.
- [ ] **US-3.** Як власник, я хочу бачити свою серію днів поспіль (🔥), що росте при виконанні, щоб відчувати прогрес і не хотіти її втрачати. **AC:** стрік++ при першому `task_completed` за день; мікросвято на 3 і 7 днів; події `streak_incremented{n}`, `streak_milestone{n}`.
- [ ] **US-4.** Як власник, який пропустив день, я хочу гуманну поведінку стріку (заморозка/новий старт без сорому), щоб не кинути апку через одну помилку. **AC:** 1 freeze-токен (поповнюється щотижня) рятує серію за пропуск; інакше м'який «почни нову серію» без різкого «−усе»; події `streak_frozen`/`streak_reset`.
- [ ] **US-5.** Як команда, ми хочемо все за A/B-флагом `daily_focus`, щоб виміряти ефект. **AC:** `getVariant('daily_focus')` ділить нових 50/50; control не бачить активного стріку/нагадування; усі події тегуються варіантом.

### Should Have
- [ ] **US-6.** Перенесення часу нагадування в налаштуваннях. **AC:** зміна часу → переплановує нотифікацію.
- [ ] **US-7.** «Тиха» поведінка: не нагадувати, якщо завдання дня вже виконані. **AC:** скасувати нагадування після `day_completed`.

## Functional Requirements

| ID | Вимога | Пріоритет | Source |
|---|---|---|---|
| FR-001 | Контекстний opt-in + вибір часу нагадування після першої перемоги | Must | US-1 |
| FR-002 | 1 локальне нагадування/день у обраний час, текст під план | Must | US-2 |
| FR-003 | Deep-link із нагадування на «Сьогодні» | Must | US-2 |
| FR-004 | Стрік = послідовні дні з ≥1 виконаним завданням; ++ при першому за день | Must | US-3 |
| FR-005 | Мікросвята на 3 і 7 днів | Must | US-3 |
| FR-006 | Гуманний стрік: 1 freeze-токен/тиждень; без різкого обнулення | Must | US-4 |
| FR-007 | A/B `daily_focus` (control vs variant), події тегуються варіантом | Must | US-5 |
| FR-008 | Зміна часу нагадування в налаштуваннях | Should | US-6 |
| FR-009 | Не нагадувати, якщо день уже виконано | Should | US-7 |

## Non-Functional Requirements

| Категорія | Вимога |
|---|---|
| Humane (constitution III) | Без покарань: пропуск не «обнуляє все»; ≤1 нагадування/день; легко вимкнути |
| Accessibility | Стрік/мікросвято — не лише колір (іконка+число+текст); `prefers-reduced-motion` вимикає конфеті |
| Analytics | Нові події + тег `variant`; сумісні зі схемою фундаменту |
| Performance | Стрік рахується локально миттєво; нагадування не блокує UI |
| Privacy | Час нагадування — локально; без нових PII |

## Consumes (from foundation Consumer Contract)
- Події: `task_completed`, `day_completed`, `app_opened`, `notification_permission`.
- Сутності: `Completion` (для підрахунку стріку), `Task`.
- Поверхня: слот «Сьогодні» (стрік-чіп уже зарезервований у фундаменті як 🔥0).
- Хук: `getVariant('daily_focus')`.

## Acceptance Criteria (feature-level)
1. Variant вмикає активний стрік + щоденне нагадування; control — ні.
2. Стрік коректно росте/заморожується/скидається за правилами FR-004/006.
3. Нагадування шле 1/день, відкриття веде на «Сьогодні».
4. Усі нові події емітяться з тегом `variant`.
5. Гуманність: один пропуск не обнуляє серію (freeze); конфеті поважає reduced-motion.
6. Опт-аут нагадувань доступний у 1 тап.
