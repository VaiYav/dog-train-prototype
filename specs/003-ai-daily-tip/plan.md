# Plan / ТЗ: AI «порада дня» (`003-ai-daily-tip`) — lite

**Spec**: [spec.md](./spec.md) · **Mode**: lite

## UX/UI (шаблон)
```
┌───────────────────────────────┐
│ 💡 Порада дня для Рекса         │
│    [AI · персоналізовано]   ↻   │
│ ─────────────────────────────  │
│ Рекс закрив 4 дні поспіль —    │
│ час ускладнити прогулянку:     │
│ додай 5 хв або нову команду.   │
│ 👍 Корисно   👎     (у проді — LLM)│
└───────────────────────────────┘
```
Розташування: **угорі «Сьогодні»**, над списком завдань.

## Як працює (логіка)
**Prod:**
```
PersonalizationContext(userId) = { dog, recentCompletions, skipped, streak, goal }
  → prompt («дай 1 коротку пораду під цей контекст, без діагнозів»)
  → LLM (cheap) → 1–2 речення → кеш на день
events: ai_tip_shown / ai_tip_refreshed / ai_tip_feedback{helpful}
A/B: getVariant('ai_daily_tip') — control без картки
```
**Прототип (офлайн):** функція `aiTipText()` детерміновано складає пораду з тих самих даних (порода-група/вік/ціль/прогрес) — імітує LLM, демонструє персоналізацію.

## Метрики
- **Primary:** D2/D7 retention (tip vs control) — контент-тригер повернення.
- **Guardrails:** 👎-rate не зростає; opt-out/uninstall стабільні; LLM cost/latency у межах; **0 медичних тверджень** (ручний + правило-фільтр).

## Guardrails AI (constitution III + безпека)
Грунтування на даних собаки; без діагнозів («не впевнений → до ветеринара»); кеш+ліміт токенів; приватність (дані лише цього користувача).

## Effort
~1–1.5 л-тижні поверх `ai-personalization-context` (enabler робить контекст; ця фіча — тонка обгортка + UI).
