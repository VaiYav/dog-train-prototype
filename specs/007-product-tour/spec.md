# Spec: Product Tour (`007-product-tour`) — lite · NON-AI

> Product Forge feature · mode: lite · type: end-user · **без AI/LLM** (детерміновані coach marks)
> Depends on: [`000-core-infrastructure`](../000-core-infrastructure/spec.md) (поверхні Today/Care/Progress, аналітика, `getVariant`)
> A/B: `getVariant('product_tour')`

## Що це (термінологія)
**Product tour / guided walkthrough** — короткий покроковий гайд по ключових екранах через **coach marks** (підсвітка елемента + спливаюча підказка). Це **не** первинний онбординг (профіль→план), а **навчання користуванню** функціоналом. Підвид in-app guidance; реалізація **детермінована** (послідовність кроків, показ раз, перезапуск), без AI.

## Гіпотеза
> **Як** новий користувач, який не одразу розуміє, що робити в застосунку, **я хочу** короткий тур по ключових екранах, **щоб** швидко зрозуміти цінність і не загубитися в перші хвилини.

## ⚠️ Свідома напруга (чесно)
Тур **може зашкодити time-to-value**, якщо блокує першу перемогу. Тому за дефолтом: показуємо **після першої перемоги** (або робимо повністю skippable, non-blocking), ≤4–5 кроків, перезапуск із налаштувань. Час показу (одразу vs після перемоги) — кандидат на A/B.

## User Stories (Must)
- [ ] **US-1.** Побачити короткий тур по ключових екранах. **AC:** ≤4–5 кроків (coach marks), підсвітка елемента + текст; події `tour_started`, `tour_step{n}`.
- [ ] **US-2.** Пропустити будь-коли. **AC:** «Пропустити» на кожному кроці; `tour_skipped`; тур не блокує дії.
- [ ] **US-3.** Показ один раз + перезапуск. **AC:** після завершення не нав'язується (persisted); кнопка «?» / налаштування перезапускають; `tour_completed`.
- [ ] **US-4.** Не шкодити активації. **AC:** дефолт — після першої перемоги; не затримує перший `task_completed`.

### Should
- [ ] **US-5.** Контекстні підказки (just-in-time) для нових фіч (coach mark при першій появі). **AC:** показ при першому контакті, не пачкою.

## Acceptance
1. Тур пояснює ключові поверхні (Сьогодні / Догляд / Прогрес / AI-елементи) за ≤5 кроків.
2. Skippable, non-blocking; показ раз; перезапуск доступний.
3. Не затримує першу перемогу (показ після неї або повністю опційний).
4. Усі події емітяться; у control туру немає.

## Metrics
**Primary:** активація / feature adoption (tour vs no-tour) + tour completion rate. **Secondary:** D1/D2 retention. **Guardrails:** time-to-value не гірше control; skip-rate; drop-off під час туру; uninstall.

## Consumes
Поверхні «Сьогодні/Догляд/Прогрес» фундаменту · аналітика (`trackEvent`) · `getVariant('product_tour')`. **Без** `PersonalizationContext`/LLM — це детермінована фіча.

## Implementation (за користувачем)
Послідовність кроків (selector + текст) → підсвітка елемента (ring) + картка з «Далі/Пропустити» + крапки прогресу → `tourSeen` у стані (показ раз) → перезапуск кнопкою «?». У проді — нативні coach marks або DAP (Pendo/Appcues). `implement: pending`.
