# Spec: «Запитай про Рекса» (`002-ai-ask-rex`) — lite

> Product Forge · lite · RICE-lite: **210** · Source: [AI backlog F1](../../docs/product/02-ai-ideas-backlog.md)
> Depends on: [`ai-personalization-context`](../ai-personalization-context/README.md) + curated KB · A/B: `getVariant('ask_rex')`

## Гіпотеза
> **Як** власник із питаннями про собаку, **я хочу** запитати в чаті («чому Рекс гавкає вночі?») і отримати відповідь **під мого песика**, **щоб** не гуглити й довіряти застосунку щодня.

## Що і де
Чат **«💬 Запитай про Рекса»** (плаваюча кнопка → оверлей). Підказані питання + вільний ввід. Відповідь **grounded** на `PersonalizationContext` (порода/вік/активність + останні дії) + куратована база порад.

## ⚠️ Безпека (критично)
**Ніколи не діагностуємо.** Медичні питання → загальна порада + **«не впевнений → до ветеринара»**. Дисклеймер у чаті. Це головний guardrail фічі (constitution III).

## User Stories (Must)
- [ ] **US-1.** Поставити питання й отримати персональну відповідь. **AC:** відповідь згадує контекст собаки; подія `ai_qa_question{topic}`.
- [ ] **US-2.** Підказані питання (швидкий старт). **AC:** 3–5 чипів під сегмент.
- [ ] **US-3.** Безпечні межі. **AC:** медичні теми → дисклеймер + ескалація; жодних діагнозів/доз.

## Acceptance
1. Відповідь явно персональна (контекст собаки), не generic.
2. Медичні питання → дисклеймер «до ветеринара», без діагнозу.
3. Подія на кожне питання; у control чату немає.

## Metrics
**Primary:** D2/D7 retention (ask vs control) + питань на користувача. **Guardrails:** 👎/скарги, 0 медичних діагнозів, LLM cost/latency.

## Consumes
`PersonalizationContext` · куратована KB · `getVariant('ask_rex')`.

## Prod vs прототип
**Prod:** PersonalizationContext + KB → prompt → LLM (з safety-фільтром). **Прототип:** `askRex(q)` повертає grounded відповідь за ключовими темами на даних собаки (демо; медичні → дисклеймер).
