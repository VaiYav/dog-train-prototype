1. Контекст продукту

DogTrain закриває для власника собаки чотири задачі в одному місці:

  • персоналізовані рекомендації щодо догляду;
  • трекінг прогулянок, активності та виконаних завдань;
  • нагадування про важливі події (вакцинації, ветеринар, грумінг);
  • індивідуальні плани тренувань і розвитку навичок.

Базовий цикл користувача (core loop), який уже описаний у сценарії:

Встановлення → Профіль собаки (порода, вік, вага, активність)
   → Онбординг-опитування → Персональний план догляду й активностей
   → Щоденні відмітки виконаних завдань → Відстеження прогресу

2. Проблема, яку розв'язуємо

За останній місяць понад 80% нових користувачів припиняють активно користуватись застосунком на 2–3 день, не завершивши перший тижневий план.

Це класична проблема активації та ранньої утримуваності (early retention), а не залучення. Користувачі доходять до цінності, але не закріплюють звичку. Гіпотези про корінь проблеми (перевантаження тижневим планом, відсутність тригера повернення, відкладена «перша перемога») перевіряються у Фазі 2.

4. Підхід: Product Forge + SpecKit

Ми не пишемо «простирадло тексту», а проганяємо роботу через Product Forge — оркестратор повного циклу, що передає фічу від ідеї до коду з людським гейтом після кожної фази. Кожна фаза дає конкретний артефакт:

Phase 0  Problem Discovery   → problem-statement.md, interview-script.md   [гейт: Go / No-Go]
Phase 1  Research            → research/*.md                                [гейт: approve]
Phase 2  Product Spec        → product-spec/README.md                       [гейт: approve]
Phase 3  Revalidation        → review.md (APPROVED)                         [гейт: approve]
Phase 4  Bridge → SpecKit    → spec.md                                      [гейт: approve]
Phase 5  Plan                → plan.md                                      [гейт: approve]
Phase 5B Tasks               → tasks.md                                     [гейт: approve]

Режими (modes): lite / standard / v-model. - Мінімальна інфраструктура (Фаза 1 цього плану) проганяється в standard mode — щоб показати повний ланцюг трасування Research → Spec → Plan → Tasks. - Retention-MVP (Фаза 3) — теж standard, бо для нього критичні метрики, гейти й експеримент.

Чому не одразу код: 80% відтоку — це гіпотеза про поведінку, а не про відсутність функції. Спочатку валідуємо проблему й найдешевший важіль, потім будуємо тільки те, що валідуємо.

5. Фазовий роадмеп

Фаза 0 — Майстер-план ✅ (цей документ)

Орієнтир, структура репозиторію, конфіг Product Forge.

Фаза 1 — Мінімальна інфраструктура 🔄 (зараз)

Базовий продукт DogTrain як фундамент, до якого кріпляться retention-фічі. Проганяємо Product Forge над specs/000-core-infrastructure/: - Phase 0 Problem Discovery — валідація кореневого JTBD власника собаки → problem-discovery/ - далі по гейтах: Research → Product Spec → Bridge (spec.md) → Plan → Tasks.

Скоуп інфраструктури (MVP-фундамент): профіль собаки, онбординг-опитування, генерація плану з шаблонів (без AI/ML), щоденні завдання + відмітки, базовий трекінг прогресу, продуктова аналітика (івенти).

Фаза 2 — Аналіз відтоку (Q1–Q3)

docs/product/01-churn-analysis.md: івенти Day1/Day2 для порівняння когорт, 5 гіпотез із методами перевірки, пріоритизація RICE, обґрунтування Confidence топ-фічі. Тут ми «думаємо по кожній фічі».

Фаза 3 — MVP retention-фічі (топ-RICE)

specs/001-daily-focus/: повний Product Forge/SpecKit-пакет на обрану фічу + технічне завдання (user story, UX/UI, логіка, метрики, acceptance) + клікабельний HTML-прототип + To-Do команди на 2 тижні. Дизайн A/B-експерименту для валідації.

Фаза 4 — Валідація та сторінка-хаб

HTML-версії всіх звітів, docs/index.html як єдина точка входу, перевірка математики RICE, узгодженості артефактів і працездатності прототипу.

Фаза 5 — AI/персоналізація track (беклог + перша фіча)

docs/product/02-ai-ideas-backlog.md: 5 згрумлених AI/LLM-фіч (Q&A «Запитай про Рекса», порада дня, місії, швидкий лог, адаптація плану) — усі персоналізовані на даних користувача, поверх спільного PersonalizationContext. Зареєстровано в config як 002–006 + enabler.

Першу AI-фічу вже дофорджено: 003-ai-daily-tip («AI порада дня», топ RICE-lite 731) — lite-forge + вбудовано в основний прототип як картка «💡 Порада дня для Рекса» угорі «Сьогодні» (у прототипі LLM імітовано правилами на даних собаки+прогресі; у проді — LLM на PersonalizationContext). Деталі — specs/003-ai-daily-tip/.

Обмеження Без кастомних AI/ML рішень, якщо їх не можна зробити силами команди за 2 тижні

Ці обмеження — головний фільтр пріоритизації: фіча, яку не можна валідувати за 2 тижні наявною командою, програє в RICE за рахунок Effort/Confidence.

8. Робочі домовленості

  • Мова всіх deliverables — українська; UI-тексти прототипу — українською.
  • Формати звітів — Markdown + HTML (HTML генерується з MD для верності змісту).
  • Структура — жодних звітів у корені; усе розкладено по теках за призначенням.
  • Тон — чесний, без прикрашання; припущення позначаються явно (через відсутність реальних даних DogTrain усі числа в RICE/метриках — це позначені оцінки, а не факти).

Примітка щодо інструментів. У глобальних інструкціях вказано використовувати MCP Serena та MCP sequential-thinking. У цьому середовищі ці MCP-сервери не підключені, тож структуроване міркування виконується вбудованими засобами, а навігація по коду — стандартними Read/Grep/Glob. Це не впливає на зміст артефактів, але чесно зазначаю розбіжність.
